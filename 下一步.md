# 下一步：从“功能已经很多”走向“系统真正会成长”

日期：2026-06-19

## 产品判断

cark 不是带 AI 的论文阅读器。

它是一个持续理解用户、积累研究上下文，并最终替用户完成研究工作的智能研究系统。

判断标准只有一个：**下一次回答，是否真的比上一次更懂用户，而且每个结论都能回到证据。**

## 本轮已经完成

### 1. 数据安全基线

- JSON 数据采用同目录临时文件、`fsync` 和原子替换。
- 覆盖写入前保留 `.bak`，主文件损坏时自动回退读取备份。
- 记忆、批注、Copilot 运行记录和资料库元数据写入 `schemaVersion`。
- 长 ID 和不安全 ID 映射为稳定短目录名；旧目录迁移采用只复制、不删除、不覆盖。
- 真实 SQLite 数据库完整性检查通过：48 篇论文、4 条阅读状态均可读取。

### 2. 搜索与稳定定位

- 搜索已进入 SQLite FTS5 持久索引，不再只依赖进程内扫描。
- 论文、正文、批注和记忆结果统一携带稳定定位器。
- 正文结果可以跳到命中内容并高亮；定位优先使用块 ID，失败时回退到引文和上下文。
- 真实数据已完成索引重建：48 篇论文生成 144 条索引，中英文查询均能返回可定位结果。

### 3. 记忆数据模型

- 建立工作、论文、全局三层记忆字段。
- 记忆支持来源、证据、置信度、候选/激活状态、派生关系、冲突关系和修订历史。
- 候选记忆不会进入智能体行为上下文；只有激活记忆可以影响回答。
- 批注可以沉淀为论文记忆，全局记忆可以注入 Copilot 上下文。

### 4. 可靠性与初步拆分

- Copilot 运行记录持久化，支持恢复、取消、失败重试和超时收口。
- 服务端已抽出搜索、定位、记忆、资料库、导出和 Copilot 运行模块。
- 前端已抽出 Copilot 运行 Hook 和稳定定位工具。
- 当前质量门：前端 44 项测试、后端 65 项测试、Lint、Python 编译和生产构建全部通过。

## 当前缺口

方向正确，但还没有完成。

1. `gui_server.py` 约 2815 行、`ReaderPage.tsx` 约 1500 行、`CommentLane.tsx` 约 1200 行。只是抽出了部分能力，核心编排仍然过重。
2. 记忆有了正确字段，但还没有真正执行“事件 → 候选 → 去重 → 冲突检测 → 置信度判断 → 激活”的决策流水线。
3. FTS5 已持久化，但当前仍以全量重建为主，需要改为按论文增量更新并记录索引版本。
4. 还没有全局提问入口，也没有跨论文证据编排和连续追问。
5. Markdown 只有单次导出，没有导出历史、版本差异和可复用研究产物模型。

## 新开发计划

### 第一阶段：拆掉三个巨型模块，2–4 天

这是下一次开发的唯一入口。停止往巨型文件继续塞功能。

服务端：

- 将 HTTP 路由按 app、papers、annotations、memory、search、copilot、exports、zotero 分组。
- `gui_server.py` 只保留启动、依赖装配和请求分发。
- 搜索索引改为按论文内容版本增量更新，不再在普通列表刷新时全量重建。

阅读页：

- 抽出论文加载、阅读状态、选区、稳定定位、批注操作和记忆操作 Hook。
- 阅读页只负责布局和能力编排。

评论区：

- 抽出线程卡片、回复编辑器、智能体运行状态和运行结果组件。
- 评论区只负责列表和线程状态编排。

验收：

- 行为和 API 完全兼容。
- 三个原巨型模块各自降到约 500 行以内。
- 所有质量门继续通过，并补充路由、定位失败回退和组件交互测试。

### 第二阶段：完成记忆决策流水线，3–5 天

实现统一入口：

`交互事件 → 候选生成 → 内容归一化 → 去重 → 冲突检测 → 置信度评分 → 激活或待审计`

具体要求：

- 每条记忆必须保存来源事件和原始证据。
- 高相似内容合并为修订，不重复制造记忆。
- 冲突内容并存，建立 `conflictsWith`，绝不静默覆盖。
- 低置信内容保留为候选，不进入智能体上下文。
- 用户保留、修改、归档和删除行为成为后续置信度校准信号。
- UI 只提供候选审计、冲突处理和修订历史，不做复杂手工分类。

验收：

- 对同一事实重复输入不会生成重复激活记忆。
- 相互矛盾的输入可以看到完整演变过程。
- 智能体回答能明确列出本次使用的记忆及其来源。

### 第三阶段：做一个真正的全局提问入口，5–7 天

首页只增加一个入口：

> 问你的全部论文、批注、记忆和研究历史

第一版只做完整纵向闭环：

- 自动判断当前论文或全局范围。
- 从论文、批注、记忆和历史研究结论中检索证据。
- 回答中的每条关键结论带可点击引用。
- 引用一步跳到论文段落、批注或记忆。
- 支持连续追问并显示本次使用的长期记忆。
- 回答可以沉淀为候选项目结论，由记忆流水线决定是否激活。

验收：没有证据的结论不显示为确定事实；任何引用都能一步回到原文。

### 第四阶段：研究产出，1–2 周

按价值顺序开发：

1. Markdown 导出历史、版本差异和可恢复版本。
2. 多论文对比和主题综述。
3. 带证据引用的研究草稿。
4. DOCX 导出。
5. 项目进度与研究驾驶舱。

## 明确不做

- 不做花哨但没有决策价值的数据驾驶舱。
- 不做复杂的手工记忆分类页面。
- 不做没有证据引用的万能聊天框。
- 不继续在三个巨型文件上堆功能。
- 不为了技术感过早引入向量数据库；FTS5 和稳定定位器先把证据闭环做扎实。
